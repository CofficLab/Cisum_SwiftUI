# /learn - 提取可重用模式

分析当前会话并提取值得保存为技能的模式。

## 触发条件

在会话期间任何时候解决了非平凡问题时运行 `/learn`。

## 要提取的内容

查找：

1. **错误解决模式**
   - 发生了什么错误？
   - 根本原因是什么？
   - 如何修复的？
   - 这对类似错误可重用吗？

2. **调试技术**
   - 非显而易见的调试步骤
   - 有效的工具组合
   - 诊断模式

3. **变通方法**
   - 库的怪癖
   - API 限制
   - 特定版本的修复

4. **项目特定模式**
   - 发现的代码库约定
   - 做出的架构决策
   - 集成模式

## 输出格式

在 `~/.claude/skills/learned/[pattern-name].md` 创建技能文件：

```markdown
# [描述性模式名称]

**提取时间：** [日期]
**上下文：** [何时适用的简短描述]

## 问题
[这解决了什么问题 - 具体说明]

## 解决方案
[模式/技术/变通方法]

## 示例
[代码示例（如适用）]

## 何时使用
[触发条件 - 什么应该激活此技能]
```

## 流程

1. 审查会话以查找可提取的模式
2. 识别最有价值/可重用的见解
3. 起草技能文件
4. 在保存前请求用户确认
5. 保存到 `~/.claude/skills/learned/`

## 注意事项

- 不要提取琐碎的修复（拼写错误、简单语法错误）
- 不要提取一次性问题（特定 API 中断等）
- 专注于将在未来会话中节省时间的模式
- 保持技能专注 - 每个技能一个模式
